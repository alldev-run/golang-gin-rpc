# pkg/db

`pkg/db` 提供数据库访问与治理相关能力，覆盖：

- 多数据库客户端封装
- 工厂模式创建数据库实例
- ORM / 查询构建 / 分页
- 连接池与池级治理
- 读写分离与代理能力
- 数据库迁移
- 指标采集、慢查询观测
- SQL 风险防护
- 数据库访问熔断能力

它适合作为业务开发中的统一数据库基础层，用于支撑常规 CRUD、分页查询、连接管理、迁移与观测等日常数据库开发需求。

## 目录结构

- `config.go`
  - 数据库配置定义
- `factory.go`
  - 统一数据库工厂与实例创建入口
- `mysql/`
  - MySQL 相关封装
- `postgres/`
  - PostgreSQL 相关封装
- `mongodb/`
  - MongoDB 相关封装
- `clickhouse/`
  - ClickHouse 相关封装
- `orm/`
  - ORM、查询构建、分页、扫描等能力
- `pool/`
  - 基础连接池封装
- `poolcb/`
  - 带熔断能力的连接池
- `poolrw/`
  - 读写池能力
- `rwproxy/`
  - 读写代理能力
- `migration/`
  - 数据库迁移能力
- `metrics/`
  - 数据库指标采集
- `slowquery/`
  - 慢查询观测
- `sqlprevention/`
  - SQL 风险防护
- `circuitbreaker/`
  - 数据库侧熔断能力

## 适用场景

`pkg/db` 当前适合：

- 常规业务系统数据库开发
- CRUD 与分页查询
- MySQL / PostgreSQL / MongoDB / ClickHouse 接入
- 基础连接池管理
- 读写分离开发
- 上线前迁移与数据库观测接入
- 慢查询监控与基础 SQL 防护

## 核心能力

### 1. 数据库工厂

通过工厂模式统一创建数据库实例，适合在 bootstrap 阶段按配置加载。

适合：

- 按配置初始化数据库资源
- 在多数据库场景下统一入口
- 屏蔽具体数据库初始化差异

### 2. ORM / 查询构建

`orm/` 提供常见数据库开发所需能力：

- 查询构建
- 分页
- 结构体扫描
- 基础 SQL 组合

适合：

- 管理后台
- 常规服务查询接口
- 列表分页与过滤查询

### 3. 连接池与读写分离

连接池相关子包包括：

- `pool`
- `poolcb`
- `poolrw`
- `rwproxy`

适合：

- 基础连接池管理
- 池级别保护
- 读写分离的业务开发

### 4. 可观测性与治理

数据库相关治理与观测能力包括：

- `metrics`
- `slowquery`
- `sqlprevention`
- `circuitbreaker`

适合：

- 暴露数据库指标
- 观测慢查询
- 基础 SQL 风险拦截
- 故障时的保护性熔断

### 5. 迁移

`migration/` 适合做：

- schema 变更
- 初始化脚本执行
- 上线前迁移管理

## 日常开发建议

如果你的目标是**日常数据库开发**，推荐优先这样使用：

- 常规业务查询与分页：`orm/`
- 基础数据库实例创建：`factory.go`
- 多数据库接入：`mysql/`、`postgres/`、`mongodb/`、`clickhouse/`
- 连接池：`pool/`
- 读写分离：`poolrw/` 或 `rwproxy/`
- 迁移：`migration/`
- 观测：`metrics/` + `slowquery/`
- 风险控制：`sqlprevention/`

## 当前状态判断

按当前仓库状态，`pkg/db` 已经**基本满足日常开发中的数据库相关操作**，包括：

- 常规增删改查
- 分页
- 多数据库接入
- 基础连接池
- 读写分离
- 数据库观测
- 慢查询与 SQL 基础防护

如果你的目标是：

- 常规业务系统开发
- 中小型服务数据库层封装
- 联调、灰度、上线准备

那么当前 `pkg/db` 是可用的。

## 当前保留项

当前仍建议保留预期，不直接把 `pkg/db` 定义为完全封顶的平台级数据库基础设施。还可以继续加强的方向包括：

- 更复杂的主从切换与故障编排
- 更成熟的分布式事务治理
- 更细粒度的数据库审计能力
- 更复杂的多租户数据库治理
- 更强的高并发数据库访问策略与压测验证

## 回归建议

修改 `pkg/db` 后，建议至少执行：

```bash
go test ./pkg/db/...
```

如果改动影响了更广的业务基础设施层，建议再执行：

```bash
go test ./pkg/...
```

## 总结

`pkg/db` 当前已经可以作为业务开发中的数据库基础层使用，适合支撑：

- 日常数据库开发
- 常规生产开发准备
- 联调与回归阶段的数据库访问能力

但如果目标是高要求的平台级数据库治理，还建议继续做后续加固。
